# 项目文档体系

> 五份文档构成完整的项目生命周期文档链，从商业论证到最终验收，环环相扣。

---

## 文档关系全景

```
                    ┌─────────────┐
                    │  1. 商业计划书 │  ← WHY（为什么做）
                    │  商业可行性   │
                    └──────┬──────┘
                           │ 输出：立项依据、预算、里程碑
                           ▼
                    ┌─────────────┐
                    │  2. 项目计划书 │  ← HOW（怎么做）
                    │  执行蓝图     │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
     ┌──────────────┐ ┌─────────┐ ┌──────────┐
     │ 3. 实施手册   │ │ 研发过程 │ │ 交付产出  │  ← DO（做出来）
     │ 执行操作指南  │ │         │ │          │
     └──────────────┘ └────┬────┘ └────┬─────┘
                           │           │
                           ▼           ▼
                    ┌─────────────┐
                    │ 4. 产品使用手册│  ← USE（用起来）
                    │ 用户操作指南   │
                    └──────┬──────┘
                           │
                           ▼
                    ┌─────────────┐
                    │ 5. 验收意见书 │  ← CHECK（验明白）
                    │ 正式验收关闭   │
                    └─────────────┘
```

## 文档依赖与数据流

| 上游文档 | 输出内容 | 下游文档 | 接收用途 |
|---------|---------|---------|---------|
| **1. 商业计划书** | 项目目标、预算、里程碑时间线 | 2. 项目计划书 | 作为项目计划的约束条件 |
| **1. 商业计划书** | 商业模式、产品定位 | 4. 产品使用手册 | 用户价值描述参考 |
| **2. 项目计划书** | 技术方案、WBS、质量标准 | 3. 实施手册 | 实施步骤的输入依据 |
| **2. 项目计划书** | 里程碑、验收标准 | 5. 验收意见书 | 验收标准的基准 |
| **3. 实施手册** | 上线步骤、运维配置 | 4. 产品使用手册 | 管理员操作部分参考 |
| **3. 实施手册** | 部署记录、测试报告 | 5. 验收意见书 | 验收证据 |
| **4. 产品使用手册** | 功能清单、操作流程 | 5. 验收意见书 | 功能验收对照依据 |

## 各文档定位

| # | 文档 | 核心问题 | 读者 | 编写阶段 | 核心章节 |
|---|------|---------|------|---------|---------|
| 1 | 商业计划书 | **为什么做？值不值得？** | 投资人/管理层 | 立项前 | 执行摘要、市场分析、商业模式、财务预测 |
| 2 | 项目计划书 | **怎么做？多久做完？** | 项目团队/管理层 | 立项时 | 技术方案、WBS、风险管理、质量标准 |
| 3 | 实施手册 | **每一步怎么操作？** | 开发/运维团队 | 执行中 | 环境搭建、部署步骤、回滚方案 |
| 4 | 产品使用手册 | **用户怎么用？** | 终端用户/管理员 | 交付前 | 功能操作、FAQ、系统管理 |
| 5 | 验收意见书 | **做得对不对？能不能结项？** | 甲乙双方 | 交付时 | 验收结果、结论签章 |

## 跨文档一致性检查清单

在项目关键节点，应检查以下跨文档一致性：

### 立项阶段（1 → 2）
- [ ] 项目计划书的目标与商业计划书的商业目标一致
- [ ] 项目计划书的预算不超出商业计划书的融资/预算额度
- [ ] 里程碑时间线与商业计划书的路线图对齐

### 执行阶段（2 → 3）
- [ ] 实施手册的技术方案与项目计划书一致
- [ ] 实施手册的环境要求与技术选型匹配
- [ ] 测试标准不低于项目计划书的质量目标

### 交付阶段（2/3 → 4）
- [ ] 使用手册的功能范围 = 项目计划书的交付物范围
- [ ] 使用手册的操作流程与实际系统一致
- [ ] 文档版本与系统版本匹配

### 验收阶段（全 → 5）
- [ ] 验收标准来源：项目计划书第4.1节（里程碑验收标准）
- [ ] 功能验收对照：产品使用手册的功能清单
- [ ] 文档验收清单：实施手册、使用手册等完整交付
- [ ] 遗留问题：不超过项目计划书风险登记册中的预案范围
