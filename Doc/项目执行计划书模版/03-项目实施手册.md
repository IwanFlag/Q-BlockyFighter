## 1. 概述

### 1.1 编写目的
本手册旨在为Q版方块人大乱斗项目的开发团队提供标准化的实施操作指南，涵盖环境搭建、代码规范、部署流程、测试流程等关键环节，确保项目按照既定计划和质量标准顺利交付。

### 1.2 适用范围
本手册适用于 Q版方块人大乱斗 项目从V0.4 Unity迁移阶段到V1.0 Steam上线的全部实施工作。

### 1.3 术语定义
| 术语 | 定义 |
|------|------|
| 帧同步 | 将所有客户端的操作输入在服务端统一按帧分发，保证所有客户端计算结果一致的技术方案 |
| 逻辑帧 | 游戏逻辑的最小计算单位，本项目为20fps（50ms/帧） |
| 渲染帧 | 画面渲染的帧率，目标60fps |
| 连招 | 通过轻重攻击组合释放的连续技能动作 |
| 弹反 | 在精确时机格挡成功，弹开对手并触发反击窗口 |
| 霸体 | 部分重攻击/大招附带的不被打断效果 |
| 体力 | 角色的行动资源，攻击/闪避/格挡均消耗体力 |

## 2. 实施总体策略

### 2.1 实施方法论
采用迭代增量开发模式，按V0.4→V0.5→V0.6→V1.0版本迭代推进，每个版本包含完整的开发→测试→验收周期。

### 2.2 实施阶段划分
```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ V0.4     │ → │ V0.5     │ → │ V0.6     │ → │ V1.0     │
│ Unity迁移│   │ 全内容    │   │ 打磨+商业化│   │ Steam上线│
└──────────┘   └──────────┘   └──────────┘   └──────────┘
```

### 2.3 实施原则
1. **帧同步优先：** 所有战斗逻辑必须通过帧同步验证，保证多人一致性
2. **先跑通再优化：** 先确保功能可运行，再优化性能和体验
3. **数据驱动：** 角色/武器/技能数值尽量数据化配置，方便平衡调整
4. **模块化：** Core/Network/Modes/AI/UI/Map/Utils 分离，降低耦合

```csharp
// 命名规范：
// - 类名：PascalCase (如 CharacterController)
// - 方法名：PascalCase (如 TakeDamage)
// - 变量名：camelCase (如 currentHealth)
// - 常量：UPPER_SNAKE_CASE (如 MAX_HEALTH)
// - 私有字段：_camelCase (如 _currentHealth)
// - 接口：I开头 (如 IDamageable)

// 代码结构：
// 1. 字段 (public → private)
// 2. 属性
// 3. 事件
// 4. Unity生命周期方法 (Awake → Start → Update → OnDestroy)
```

## 6. 部署步骤

### 6.1 服务器部署

#### 开发环境
```bash
cd Server
dotnet run
# 启动后输出：
# WebSocket: ws://localhost:3000
# HTTP API: http://localhost:3000/api/status
# 帧率: 20 fps
```

#### 生产环境
```bash
# 步骤1：发布Release版本
cd Server
dotnet publish -c Release -o ./publish

# 步骤2：上传到服务器
scp -r ./publish user@server:/opt/q-blockyfighter/

# 步骤3：配置systemd服务（Linux）
# 创建 /etc/systemd/system/qbf-server.service
# [Unit]
# Description=Q-BlockyFighter Server
# [Service]
# WorkingDirectory=/opt/q-blockyfighter
# ExecStart=/usr/bin/dotnet Q-BlockyFighter.Server.dll
# Restart=always
# [Install]
# WantedBy=multi-user.target

# 步骤4：启动服务
sudo systemctl enable qbf-server
sudo systemctl start qbf-server
sudo systemctl status qbf-server
```

### 6.2 客户端打包

```
Unity Editor → File → Build Settings
→ Platform: WebGL
→ Player Settings: 配置分辨率/图标
```

#### Unity Windows（Steam版）

```
Unity Editor → File → Build Settings

→ 输出: Q-BlockyFighter.exe

```

### 6.3 关键配置参数
| 配置项 | 默认值 | 说明 |

### 7.3 帧同步一致性测试
```
测试方法：
1. 启动服务器 + 2个客户端
2. 客户端A和B执行相同输入序列
3. 每帧对比两个客户端的角色状态（位置、血量、技能CD等）
4. 如果任意一帧不一致，测试失败

测试场景：
- 基础移动
- 连招攻击
- 闪避/格挡
- 武器切换
- 技能释放
- 击杀回血
- 体力耗尽
```

### 8.3 常见问题处理
| 问题现象 | 可能原因 | 排查步骤 | 解决方案 |
|---------|---------|---------|---------|
| 客户端连接失败 | 服务器未启动/端口不通 | 检查服务器状态、端口开放 | 重启服务器、检查防火墙 |
| 帧同步不同步 | 丢帧/逻辑随机性 | 查看服务器日志、对比帧数据 | 排除随机因素、增加帧缓冲 |
| 延迟过高 | 网络带宽不足 | ping服务器、检查带宽 | 更换服务器区域 |
| 客户端卡顿 | 渲染压力大 | 查看Profiler | 降低画质/优化渲染 |
| 匹配不到人 | 在线人数少 | 查看在线统计 | 调整匹配范围、推广获客 |

## 9. 安全管理

### 9.1 安全要求
| 维度 | 要求 |
|------|------|
| 输入验证 | 服务端校验所有关键输入（移动速度、攻击频率等） |
| 客户端不信任 | 所有数值计算在服务端进行，客户端仅负责输入和渲染 |
| 反作弊 | 服务端检测异常输入（移动速度超限、攻击频率异常） |
| 数据安全 | 玩家数据加密存储，不暴露敏感信息 |

### 9.2 安全检查清单
- [ ] 服务端校验移动速度上限
- [ ] 服务端校验攻击频率上限
- [ ] 服务端校验技能冷却
- [ ] 客户端不直接修改血量/伤害等数值
- [ ] 异常行为自动踢出+记录
- [ ] 日志脱敏
